# Сборка и разработка

Эта инструкция рассчитана на Windows и PowerShell.

## Разработка (editable) — чаще всего этого достаточно

```powershell
.\scripts\dev.ps1
```

Это создаст `.venv` (если её нет), обновит `pip` и установит проект в editable-режиме.
В этом режиме изменения в коде сразу видны, пересобирать ничего не нужно.

## Сборка релизных артефактов (sdist + wheel) — нужна только для релиза

```powershell
.\scripts\build.ps1
```

Если нужно сначала очистить артефакты сборки:

```powershell
.\scripts\build.ps1 -Clean
```

Готовые артефакты появятся в `dist\`.

## Что такое файлы в dist

- `*.whl` — wheel, быстрый установочный пакет.
- `*.tar.gz` — sdist, архив исходников для установки/сборки.

## Когда build НЕ нужен

- Ты просто разрабатываешь проект локально.
- Ты запускаешь сервер из исходников.
- Тебе не нужно публиковать пакет и не нужно переносить его на другие машины.

## Когда build нужен

- Ты хочешь опубликовать пакет (например, на PyPI).
- Ты хочешь передать готовый пакет кому-то ещё.
- Ты проверяешь, что проект собирается по стандарту (PEP 517).

## Ручной способ (если нужен)

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install -e .
```

## Что принято в Python-проектах

- Описывать проект в `pyproject.toml`.
- Для разработки использовать editable-установку (`pip install -e .`).
- Для релиза собирать через `python -m build` (PEP 517).
